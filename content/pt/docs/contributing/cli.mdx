O YAMLResume adota uma estrutura de mono-repo, o que significa que todos os
pacotes estão contidos num único repositório. Graças ao [workspace
de pnpm](https://pnpm.io/workspaces), esta abordagem simplifica a gestão de
dependências e a versão entre pacotes.

No momento de escrever, o repositório do YAMLResume tem dois pacotes:

- [yamlresume/cli](https://github.com/yamlresume/yamlresume/tree/main/packages/cli):
  a interface CLI para o YAMLResume.
- [yamlresume/core](https://github.com/yamlresume/yamlresume/tree/main/packages/core):
  contém o motor central para o YAMLResume. Processa a análise, validação e
  renderização de currículos baseado no [esquema](/docs/compiler/schema)
  oficial.

## yamlresume/core build

`yamlresume/cli` depende de `yamlresume/core` para funcionar corretamente. Então
quando quiseres testar `yamlresume/cli`, precisas de assegurar que
`yamlresume/core` também está construído e disponível.

Podes fazer isto com os seguintes comandos:

```bash
# construir para propósito de desenvolvimento
pnpm core build

# construir para produção, o qual minificará a saída e gerará ficheiros
# DTS de TypeScript `.d.ts`
pnpm core build:prod

# vigiar alterações de ficheiro e construir automaticamente
pnpm core build:watch
```

## yamlresume/cli build

Uma vez que `yamlresume/core` está construído, podes executar os comandos CLI
de duas formas:

1. construir `yamlresume/cli` localmente e executar node.js com `dist/cli.js`:

```bash
pnpm cli build
node packages/cli/dist/cli.js <comando> [opções]
```

2. executar `yamlresume/cli` diretamente com a ajuda de [tsx](https://tsx.is/):

```bash
pnpm cli dev <comando> [opções]

# este comando será traduzido para
# tsx src/cli.ts
```
