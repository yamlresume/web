## Introdução

Criar um currículo pode não ser difícil, mas definitivamente é um processo
aborrecido, longo e tedioso.

Com o YAMLResume, pode criar e controlar versões dos seus currículos com YAML e
criar PDFs perfeitos ao pixel com design e composição profissional num
instante.

O YAMLResume é o motor central do [PPResume](https://ppresume.com), um
construtor baseado em LaTeX. Ajudou [milhares de
pessoas](https://blog.ppresume.com/posts/2024-recap) em todo o mundo a criar
currículos perfeitos ao pixel e de aspeto profissional e a conseguir os seus
trabalhos dos sonhos. Aqui decisámos tornar o motor em código aberto, para dar
às pessoas o direito e a liberdade de dizer [não ao bloqueio de
fornecedor](https://blog.ppresume.com/posts/no-vendor-lock-in)

Currículo](/static/images/docs/yamlresume-new-and-build.webp)

## Começar

### Utilizadores de Docker

Se é um utilizador de docker, tem sorte, temos uma [imagem de
docker](https://hub.docker.com/r/yamlresume/yamlresume) para si, que tem
[yamlresume cli](https://www.npmjs.com/package/yamlresume) instalado com todas
as suas dependências, como [XeTeX](https://www.tug.org/xetex/) e algumas [fontes
recomendadas](/pt/docs/installation#font).

Pode simplesmente criar um novo currículo com o seguinte comando:

```sh
docker run --rm -v $(pwd):/home/yamlresume yamlresume/yamlresume new my-resume.yml
```

Edite `my-resume.yml` em conformidade e depois construa o currículo de yaml
para pdf:

```sh
docker run --rm -v $(pwd):/home/yamlresume yamlresume/yamlresume build my-resume.yml
```

### Utilizadores de macOS

Assumamos que está a usar macOS com [Homebrew](https://brew.sh/) como gestor de
pacotes. Se não for o caso, por favor consulte o guia de
[Instalação](/pt/docs/installation) para instruções detalhadas para o seu sistema
operativo.

<Steps>

<Step>
#### Instalar Node.js

Precisa de [instalar
Node.js](https://nodejs.org/en/download/package-manager/all#macos) 20.x ou
mais recente.

```sh
brew install node
```

</Step>

<Step>
#### Instalar Motor de Composição

O YAMLResume precisa de um [motor de
composição](/pt/docs/installation#typesetting-engine) para gerar PDF.
Atualmente suporta dois motores de composição:

- [XeTeX](/pt/docs/installation#xetex)
- [Tectonic](/pt/docs/installation#tectonic)

<Tabs groupId="typesetting-engine" persist items={["XeTeX", "Tectonic"]}>
```sh tab="XeTeX"
brew install mactex
```

```sh tab="Tectonic"
brew install tectonic
```

</Tabs>
</Step>

<Step>
#### Instalar YAMLResume

Instalar `yamlresume` é muito fácil. Pode instalá-lo usando o seu gestor de
pacotes Node.js favorito:

<Tabs groupId='package-manager' persist items={['npm', 'pnpm', 'yarn', 'bun']}>
```sh tab="npm"
npm install -g yamlresume
```

```sh tab="pnpm"
pnpm add -g yamlresume
```

```sh tab="yarn"
yarn global add yamlresume
```

```sh tab="bun"
bun add -g yamlresume
```

</Tabs>

Também pode instalar [`yamlresume` via
Homebrew](http://formulae.brew.sh/formula/yamlresume):

```sh
brew install yamlresume
```

Verifique que `yamlresume` foi instalado com sucesso:

```console
$ yamlresume help
```

<Step>
#### [Opcional] Instalar Fontes

##### Fonte Linux Libertine

Também recomendamos instalar a [fonte Linux
Libertine](/pt/docs/installation#linux-libertine) para a melhor saída PDF de
aspeto, mas isto é opcional.

```sh
brew install font-linux-libertine
```

##### Fontes Noto

Se precisar de criar um currículo CJK (Chinês, Japonês, Coreano), recomendamos que
instale a [fonte Noto](https://fonts.google.com/noto):

```sh
brew install font-noto-sans-cjk font-noto-serif-cjk
```

</Step>

<Step>
#### Criar o Seu Primeiro Currículo YAML

OK, temos tudo pronto, agora podemos criar um currículo yaml com
`yamlresume new`:

```console
$ yamlresume new my-resume.yml
✔ Created my-resume.yml successfully.
```

Edite `resume.yml` em conformidade, depois pode gerar um PDF com `yamlresume
build`:

```console
$ yamlresume build my-resume.yml
◐ Generating resume PDF file with command: xelatex -halt-on-error my-resume.tex...
✔ Generated resumep PDF file successfully.
```

Também pode usar `yamlresume dev` para construir o currículo quando houver alterações no ficheiro:

```console
$ yamlresume dev my-resume.yml
◐ Generating resume PDF file with command: xelatex -halt-on-error my-resume.tex...
✔ Generated resume PDF file successfully.
◐ Watching file changes: my-resume.yml...
```

</Step>
</Steps>
