YAMLResume baru saja
diterima](https://github.com/Homebrew/homebrew-core/pull/246949) oleh
[Homebrew](https://brew.sh) (package manager paling populer di macOS). Itu
artinya pengguna macOS sekarang dapat memasang dan memperbarui YAMLResume
dengan satu perintah yang dapat dipercaya—tanpa instalasi global [Node.js]
(https://nodejs.org) manual, tanpa memainkan path. Postingan ini menunjukkan
langkah demi langkah yamlresume setup yang lebih detail dengan Homebrew dan
mesin typesetting untuk output PDF terbaik.

![YAMLResume on Homebrew](/static/images/blog/yamlresume-homebrew/yamlresume-homebrew.webp)

## Mengapa Ini Penting

Sebelumnya, jalur khas untuk mendapatkan CLI YAMLResume untuk pengguna macOS
adalah:

1. [pasang Node.js](/docs/installation#nodejs),
2. [pasang CLI `yamlresume`](/docs/installation#yamlresume-cli) dengan package
   manager Node.js (npm/pnpm/yarn/bun)
3. [pasang mesin typesetting](/docs/installation#typesetting-engine) untuk
   menghasilkan PDF

[Formula Homebrew](https://formulae.brew.sh/formula/yamlresume) dapat
menggabungkan langkah 1 dan 2 menjadi satu, karena Homebrew dapat mengelola
dependensi Node.js untuk Anda. Ini mengurangi masalah konfigurasi lingkungan
dan menyederhanakan upgrade (`brew upgrade yamlresume`).

## Instalasi Cepat

```sh
brew install yamlresume
```

Homebrew akan memasang Node.js sebagai dependensi jika Anda belum
memilikinya.

![brew install yamlresume](/static/images/blog/yamlresume-homebrew/brew-install-yamlresume.webp)

YAMLResume bergantung pada mesin typesetting untuk menghasilkan PDF, Anda
dapat memasang [MacTeX](/docs/installation#xetex) atau
[Tectonic](/docs/installation#tectonic) melalui Homebrew.

### Opsi A: Full TeX Live (XeTeX)

Pasang melalui MacTeX cask (termasuk XeTeX):

```sh
# install the full mactex
brew install mactex
# or if you want to save some disk space
brew install mactex-no-gui
```

Setelah instalasi, pastikan binary TeX ada di `PATH` (Homebrew biasanya
menautkannya di bawah `/Library/TeX/texbin`). Test:


### Opsi B: Tectonic (Ringan)

```console
$ brew install tectonic
$ tectonic --version
```

Mana yang harus Anda pilih?

| Mesin   | Kelebihan                                              | Kekurangan                                   |
| -------- | ---------------------------------------------------- | -------------------------------------- |
| XeTeX    | kompatibilitas maksimal; ekosistem matang              | download besar (4-6 GB)               |
| Tectonic | footprint awal kecil; auto-fetch paket yang dibutuhkan | occasional compatibility gaps vs XeTeX |

Jika Anda mengalami masalah paket LaTeX yang tidak dapat dijelaskan atau
kasus edge penanganan font lanjutan, fallback ke XeTeX.

## Membuat dan Membangun Curriculum Vitae

Seperti biasa, Anda dapat membuat curriculum vitae YAML starter dengan:

```console
$ yamlresume new my-resume.yml
✔ Created my-resume.yml successfully.
```

Periksa dan edit `my-resume.yml` di editor Anda, lalu build ke PDF:

```console
$ yamlresume build my-resume.yml
◐ Generating resume PDF with command: xelatex -halt-on-error my-resume.tex...
✔ Generated resume PDF successfully.
```

Di baliknya YAMLResume akan:

1. transform `my-resume.yml` menjadi LaTeX (`my-resume.tex`).
2. panggil mesin typesetting Anda (`xelatex` atau `tectonic`).
3. hasilkan `my-resume.pdf`.

## Upgrade

Tetap terkini (template baru, perbaikan, peningkatan i18n):

```console
$ brew update
$ brew upgrade yamlresume
```

## Uninstall

```console
$ brew uninstall yamlresume
```

Hapus dependensi opsional hanya jika Anda tidak membutuhkannya lagi (font,
mesin TeX, dll.).

## Pemecahan Masalah

Mode verbose membantu mendiagnosis masalah build curriculum vitae:

```console
$ yamlresume build --verbose my-resume.yml
```

Perbaikan umum:

- mesin typesetting hilang: pasang MacTeX atau gunakan Tectonic.
- permission denied menulis file: pastikan Anda di direktori yang dapat
  ditulis (hindari path sistem).
- ketidakcocokan mesin/kesalahan LaTeX aneh di Tectonic: fallback ke XeTeX
  (`brew install mactex`).
